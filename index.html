Welcome to Our Data Processing Project

This repository automates the processing of `data.xlsx` into structured `result.json` using a Python script, all managed through a robust CI/CD pipeline.

About This Project

Core Functionality

The heart of this project lies in `execute.py`, a Python script designed to read, process, and transform data from `data.xlsx`. It's built for efficiency and maintainability, ensuring data integrity.

Technology Stack

  • Python 3.11+
  • Pandas 2.3+
  • GitHub Actions for CI/CD
  • Tailwind CSS for responsive UI (this page)

Data Overview

The primary input data is data.xlsx. During the CI process, this file is converted to data.csv for consistent processing. The final processed output is result.json, which is then published.

Input: data.xlsx

Original data source, managed in the repository.

Intermediate: data.csv

Generated from data.xlsx during CI for processing.

Output: result.json

Final processed data, published via GitHub Pages.

View latest result.json

CI/CD Workflow

Our Continuous Integration workflow is defined in .github/workflows/ci.yml. It automates code quality checks, data processing, and result publication.

  • Linting: Ruff ensures code quality and style compliance for Python files.
  • Data Conversion: data.xlsx is converted to data.csv.
  • Execution: execute.py runs, generating result.json.
  • Publication: result.json is published as a static asset via GitHub Pages.
